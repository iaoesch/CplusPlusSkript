\section{Grundsõtzliches}

In C und \Cpp werden alle Statements (Befehle) mit einem Semikolon ';' abgeschlossen.
Variablen welche ausserhalb von Funktionen definiert werden, und Funktionen selbst sind grundsõtzlich global, das heisst sie k÷nnen von anderen Modulen benutzt oder aufgerufen werden. Um den G³ltigkeitsbereich einer Funktion oder einer globalen Variable auf
ein Modul zu beschrõnken, muss diese als static deklariert werden, dann kann nur noch innerhalb des entsprechenden Moduls auf sie zugegriffen werden. Definitionen von Funktionen innerhalb von Funktionen (lokale Funktionen) sind nicht m÷glich.  

\section{Module}
Ein C/\Cpp Programm ist normalerweise in mehrere Module (Dateien) aufgeteilt. Ein Modul besteht aus einem \OviIndex{Deklarationteil} (\OviIndex{Schnittstellenbeschreibung}) in einer \OviIndex{Headerdatei}, normalerweise mit der Dateiendung .h oder .hpp, und einem \OviIndex{Implementationsteil} in einer Datei mit der Endung .cpp (Gelegentlich auch .c++ oder .cxx).

In der Headerdatei sind alle \OviIndex{Deklarationen} von Funktionen, Variablen, Konstanten, Datentypen und Klassen enthalten, die vom Modul zur Verf³gung gestellt werden. Im Implementationsteil sind die entsprechenden \OviIndex{Definitionen} (Code und Speicherreservationen) enthalten, plus eventuell weitere lokale Variablen, Funktionen und Hilfsklassen.

In \Cpp wird ³blicherweise jede Klasse in einem eigenen Modul implementiert, eventuell zusammen mit Hilfsklassen, die nur von diesem Modul verwendet werden.
Wenn ein Modul Dienste von einem anderen Modul in Anspruch nimmt, muss es dessen Headerdatei mit dem \#include-Befehl einbinden.

Jedes Modul (.cpp) wird vom Compiler selbstõndig und unabhõngig von anderen Modulen (.cpp) Compiliert. Aus jedem Modul wird somit eine Objektdatei (Maschinencodedatei) erzeugt, welche erst vom Linker zum lauffõhigen Programm zusammengef³gt werden.

Fehlende Funktionen oder globale Variablen, oder doppelt definierte Funktionen werden deshalb meist erst vom Linker erkannt, und mit nicht sehr aufschlussreichen Fehlermeldungen geahndet ('Symbol xyz not found' oder 'Symbol xyz allready defined in Module qwert.cpp').

\section{Kommentare}
\Cpp kennt zwei Arten von Kommentaren: Den Zeilenkommentar und den Blockkommentar. Ein Zeilenkommentar beginnt mit der Zeichenfolge //, der Rest der Zeile gilt dann als Kommentar. Ein Kommentarblock wird mit /* eingeleitet und endet mit */. Ein Blockkommentar kann sich ³ber mehrere Zeilen erstrecken. Blockkommentare k÷nnen nicht verschachtelt werden.
\begin{CCode}
// Ein Zeilenkommentar

   geht ueber mehrere
   Zeilen */
\end{CCode}

\section{Prõprozessorbefehle}
Prõprozessorbefehle erkennt man an dem vorangestellten \#. Dabei handelt es sich nicht um C oder \Cpp Befehle, sondern um spezielle Prõprozessorbefehle. Diese Befehle dienen zum Verõndern und Ersetzen von Text und dem bedingten Compilieren. Nachfolgend einige oft verwendete Prõprozessorbefehle.

\begin{CCode}
#include "test.h"   // der wohl wichtigste Befehl, fuegt die angegebene Datei an 
                    // dieser Stelle in den Code ein.
#if 0               // Bedingte Kompilierung
...                 // Der Teil dessen Bedingung nicht zutrifft wird als
#else               // Kommentar behandelt, also vom Compiler ignoriert
...
#endif

#define pi 3.14     // Makro (definition einer Konstanten)

#ifdef DEBUG        // Bedingte Kompilierung
... // Wird nur compiliert wenn Makro Debug definiert ist
#endif
\end{CCode}
Speziell beim Einbinden von Headerdateien kann es geschehen, dass die selbe Datei ³ber verschiedene Wege mehrfach eingebunden wird. Das kann zu Problemen f³hren, besonders bei Klassen, weil diese nur einmal definiert werden d³rfen. Das Problem lõsst sich umgehen, wenn der Inhalt jeder Headerdatei mit folgenden Befehlen eingeschlossen wird:
\begin{CCode}

#ifndef HEADER_DATEI_NAME_HPP // ifndef -> if not defined, Gegenteil von ifdef
#define HEADER_DATEI_NAME_HPP

// Inhalt der Headerdatei,    wird nur Compiliert wenn HEADER_DATEI_NAME_HPP noch
                           // nicht definiert war, also nur beim ersten Mal
#endif
\end{CCode}

Damit wird garantiert, dass der Inhalt der Headerdatei nur einmal ber³cksichtigt wird. Diese Methode ist beinahe Standard, man findet sie bei allen Systemdateien und Bibliotheken, und die meisten Programmierer wenden sie grundsõtzlich an, um eventuellen Problemen vorzubeugen.
Es gibt zwei Formen f³r den Includebefehl:
\#include <......> sucht die Datei im Systemverzeichnis des Compilers und allfõlligen weiteren vom Benutzer angegebenen Verzeichnissen. Diese Form wird f³r Bibliotheksfunktionen verwendet.
\#include ''.....'' sucht zuerst im gleichen Verzeichnis in welchem die cpp Datei liegt, und dann in denselben Verzeichnissen wie \#include <...>. Diese Form wird f³r Projektbezogene Dateien (Vom Benutzer selbst geschriebene Dateien) eingesetzt. 

Zusõtzlich gibt es einige vordefinierte Makros, welche Informationen ³ber den Compiler und Dateien enthalten: \_\_FILE\_\_, \_\_LINE\_\_, \_\_DATE\_\_, \_\_TIME\_\_, \_\_cplusplus und weitere. \_\_cplusplus ist definiert, wenn ein C++ Compiler compiliert.

\section{Ausnahmebehandlung (Exceptions)}
Oft sind der Ort, an dem ein Fehler entsteht, und der Ort, wo er angemessen behandelt werden kann, an verschiedenen Stellen im Programm zu finden. Dies lõsst das behandeln von Fehlern meist zu einem komplizierten und fehleranfõlligen Problem werden.

Um die Fehlerbehandlung zu vereinfachen, wurde das \OviIndex{Exceptionhandling} in \Cpp eingef³hrt. Wenn irgendwo im Programmlauf ein Fehler auftritt, der an dieser Stelle nicht behandelt werden kann, wirft man mit throw() eine \OviIndex{Ausnahme}. Damit wird die normale Abarbeitung des Programmes abgebrochen und zum nõchsten passenden Fehlerbehandlungsteil gesprungen. Dieser kann innerhalb der selben Funktion sein, aber auch in einer der aufrufenden Funktionen. Dabei wird die Abarbeitung der aktuellen Funktion(-en) abgebrochen und in der Aufrufhierarchie (=Verschachtelung von Aufrufen) zur³ckgekehrt, bis ein passender \OviIndex{Handler} gefunden wird. Falls kein passender Handler existiert, wird das Programm mit einer Laufzeitfehlermeldung abgebrochen.

Mit \OviIndex{throw} werden normalerweise Objekte geworfen, es kann aber im Prinzip jeder Datentyp geworfen werden (Also auch int, char, ...). Beim Fangen wird jeweils der Handler aktiv, der den entsprechenden Datentyp auffangen kann.
Handler f³r Basisklassen fangen auch Objekte von davon abgeleiteten Klassen. Der Handler catch(...) fõngt jede beliebige Exception. Nach jedem \OviIndex{try}-Block folgt eine Liste von Handlern (catch()). Die Handler werden in der Reihenfolge ihrer Definition durchsucht,  der erste passende Handler wird benutzt, und alle weiteren Handler
dieser Liste ignoriert. Somit werden alle Handler nach \OviIndex{catch}(...) ignoriert. 

Beispiel:

};

};

};

      throw (NegativeSquareRootErrorClass());
   }
   int result = x / 2;
   // some clever square root algorithm
   return (result);

   if (x == 0) {
      throw(DivideByZerroErrorClass());
   }
   return (1/x);
}

   int Result;

      int r1 = Inverse(x1);
      int r2 = Inverse(x2);
      Result = SquareRoot(r1) + SquareRoot(r2);
      Result = Inverse(Result);
   }
   catch (DivideByZerroErrorClass &e) {

   }

   cin >> b;

   }
   catch (NegativeSquareRootErrorClass &e) {

F³r dynamisch erzeugte Objekte (new) ist der Benutzer hingegen selbst zustõndig (Durch entsprechende catch throw Anweisungen).

Die \OviIndex{STL} (\OviIndex{Standard Template Library}) ist eine der ersten Anwendungen, die umfassend von Templates gebrauch machte, und zudem die Erweiterung und Verbesserung der Templatesyntax zur Folge hatte.

\subsection{Kurz³bersicht zu Container map<> aus STL}
Die \OviIndex{map} ist ein \OviIndex{assozitiver Container}, das heisst er verwaltet Wertepaare.
Ein Wert ist der \OviIndex{Schl³ssel} des Paares, und der andere Wert beinhaltet die
Daten, welche unter diesem Schl³ssel abgelegt sind. Der Schl³ssel kann wie
der Index bei einem gew÷hnlichen Array betrachtet werden, mit dem
Schl³ssel erhõlt man Zugriff auf die dazugeh÷rigen Daten. Deshalb muss
jeder Schl³ssel innerhalb einer map einmalig sein. Die Daten werden innerhalb der
map nach dem Schl³ssel in aufsteigender Reihenfolge sortiert, und beim
Iterieren in dieser Reihenfolge abgearbeitet. Zur einfachen Benutzung
der map wurde der Indexoperator [] ³berladen, so dass der Schl³ssel direkt als
Index verwendet werden kann. Wird ein nicht existierender Schl³ssel
benutzt, erzeugt die map automatisch einen leeren Eintrag f³r diesen
Schl³ssel und liefert eine Referenz darauf zur³ck.

Der Schl³ssel muss ein Datentyp mit einer eindeutigen Ordnung sein, d. H.
die Relation '<' muss definiert und eindeutig sein. (Sonst k÷nnten die
Schl³ssel nicht sortiert werden).

Der Wert kann ein beliebiger Datentyp sein (Im Extremfall eine Liste
oder eine weitere map). 

Als Beispiel erstellen wir eine map mit Studenten und ihren Noten.

\begin{CCode}

#include<string>

void f(void) 
{  
   MyMap["Hans"]   = 5.5;        //Ein paar Eintraege...
   MyMap["Fritz"]  = 5.0;        
   MyMap["Franz"]  = 5.5;
   MyMap["Max"]    = 4.5;
   MyMap["Moritz"] = 4.5;
   MyMap["Max"]    = 5.5;        // Vorheriger Wert von Max wird ueberschrieben

   float Note = MyMap["Fritz"];        // Lesen ist auch moeglich

   map<string, double>::iterator ptr; //Definieren eines Iterators (Zeigers) in die Map

   ptr = MyMap.begin();          //Iterator auf erstes Element der Map zeigen lassen;

   while(ptr != MyMap.end()) {   // Fuer alle Elemente in der Map
    
      // Name und Note ausgeben (first und second sind durch map vorgegeben)
      cout << ptr->first << " hat die Note " << ptr->second << endl;   
      ptr++;
   }
}

\end{CCode}

Mit einer map ist es sehr einfach, Daten zu verwalten. F³r einen bestimmten Schl³ssel kann es aber nur einen Eintrag geben. Falls mehrere Eintrõge mit demselben Schl³ssel m÷glich sein sollen,
muss eine Multimap verwendet werden.

F³r die map gibt es noch weitere, hier nicht vorgestellte Operationen wie das Suchen von
Eintrõgen, hinzuf³gen von Eintrõgen oder l÷schen von Eintrõgen. (Der Indexoperator [] Sucht \emph{und} f³gt hinzu, falls nicht gefunden)
